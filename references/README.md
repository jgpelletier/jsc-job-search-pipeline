# references/

Personal context the agents read to do their work. Fill these in once at setup; update as your story evolves.

| File | Purpose |
|------|---------|
| `resume.md` | Master resume in markdown. Source of truth for bullets and metrics. |
| `cmf.md` | Candidate Market Fit — your one-line pitch, target scenarios, hard constraints. |
| `mnookin.md` | Hard must-haves and must-not signals used by `score-fit` and `analyze-jd`. |
| `stories/` | Verified work stories. One file per story. Used instead of reconstructing facts from resume bullets. |
| `analyses/` | Per-role artifacts produced during the pipeline (jd, fit, bullets, outreach, etc.). |
| `example/` | A worked example with a fictional persona (Maya Chen). Skim before filling in your own — shows what "filled in" looks like, especially for `stories/`. Safe to delete once your own files are in place. |

## Setup

If you have just cloned the template and these files are still stubs, open the project in Claude Code and say "help me set this up". The `setup` skill will interview you and write `resume.md`, `cmf.md`, `mnookin.md`, and 2–3 story files in one session.
